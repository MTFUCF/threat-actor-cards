# Contributing

Thanks for taking an interest in **Threat Actor Cards**.

This repository is part of Matthew Faber's cybersecurity portfolio, so changes should stay practical, easy to review, and easy to deploy.

## Ground rules

1. Keep the app lightweight. If the project is vanilla HTML/CSS/JS, do not add a build step.
2. Preserve the documented data shape and deployment target.
3. Favor small pull requests with before/after screenshots when UI changes.
4. Never commit secrets, API keys, or `local.settings.json`.

## Local workflow

1. Read `README.md`, `TESTING.md`, and `.github/copilot-instructions.md` before editing.
2. Run the project locally using the documented commands.
3. Complete the manual QA checklist relevant to the change.
4. Update docs when the scope or behavior changes.

## Pull request expectations

- Summarize what changed and why.
- List how you tested it.
- Attach a screenshot for visible UI changes.
- Call out any follow-up work that should stay out of the current change.
